🔧 技術的説明 — 文字化け防止 & プライバシー保護

このツールは、ZIPファイル内のテキストデータを安全に抽出し、文字化けを最小限に抑えながら出力します。すべての処理はブラウザ内で完結し、サーバーにデータを送信しません。

📄 文字化けを防ぐ技術

  • 自動判別 マルチエンコーディング自動検出 — ファイルの先頭バイト、BOM、統計情報を基に UTF-8, Shift_JIS, EUC-JP, UTF-16LE などから最適な文字コードを自動選択。ユーザーが優先順位をカスタマイズ可能。
  • フォールバック 段階的デコード — 自動判別に失敗した場合、強制エンコーディングや置換文字（�）を用いて安全に表示。エラー行をスキップするモードも用意。
  • バイナリ識別 マジックバイト & ヌルバイト検出 — 画像・動画・実行ファイルなどを即座に判別し、テキストとして無理にデコードしないよう保護。Base64・Hexダンプ・目録のみの3モードから選択可能。
  • サイズ制御 巨大ファイルの切り詰め — メモリ負荷を避けるため、閾値を超えるファイルは先頭のみ・先頭＋末尾・エラー中断から選択でき、必要以上に大きなテキストを扱わない設計。

🔒 プライバシー保護技術

  • 完全ローカル処理 — アップロードされたZIPファイルは サーバーに一切送信されず、ブラウザ上の JavaScript（unzipit ライブラリ）で処理されます。インターネット接続は広告表示と Ko-fi リンクのみに利用。
  • 安全な一時保存 — 出力データは IndexedDB に一時保存され、ダウンロード完了後に自動消去。File System Access API に対応したブラウザでは、直接ローカルファイルとして保存可能（追加の許可が必要）。
  • 除外ルール — .gitignore インポート機能により、機密情報を含むファイル（パスワード・APIキーなど）を事前に除外可能。正規表現による柔軟なフィルタリング。
  • データ最小化 — バイナリファイルはデフォルトで「目録のみ」出力し、不要なデータの埋め込みを防止。Base64/Hex は任意で有効化。

⚙️ その他の特長

  • ストリーム処理 — メモリ使用量を抑え、数GBのZIPファイルでもスムーズに動作。
  • 多言語対応 — 日本語／英語の表示切替。
  • 出力形式 — Plain Text, Markdown, JSON から選択可能。
